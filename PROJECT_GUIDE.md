# 📱 MESSENGER — РУКОВОДСТВО ПО ПРОЕКТУ

> Полное объяснение архитектуры, модулей и с чего начать разработку

---

## 📌 С ЧЕГО НАЧАТЬ

Разработку нужно вести в строгом порядке — каждый шаг зависит от предыдущего:

```
1. База данных
2. Backend (API)
3. Шифрование
4. Мобильное приложение
5. Звонки
6. Safe Mode
7. Админ панель
```

---

## 🗄️ ШАГ 1 — БАЗА ДАННЫХ

**Что делать первым делом:**
- Установить PostgreSQL
- Запустить `schema.sql` — создаст все 17 основных таблиц
- Запустить `schema_encryption_mode.sql` — добавит 3 таблицы и расширит 4 существующих

**Итого таблиц:** 20

**Почему первым:** всё остальное (backend, приложение) зависит от структуры базы данных. Без неё ничего не работает.

---

## ⚙️ ШАГ 2 — BACKEND (Серверная часть)

**Что это такое:**
Сервер который принимает запросы от приложения, работает с базой данных и отдаёт ответы.

**Технологии:** Node.js + Express / Fastify или Python + FastAPI

**Из каких модулей состоит:**

---

### 📦 Модуль `users`
**Таблицы:** `user`, `profile`, `user_key`

**Для чего нужен:**
- Регистрация нового пользователя
- Вход в аккаунт (выдача токена)
- Выход из аккаунта
- Удаление аккаунта
- Получение информации о пользователе по ID или username
- Обновление профиля (имя, фото, bio, статус)
- Смена пароля
- Управление публичными ключами (при смене устройства)
- Отображение статуса онлайн / оффлайн
- "Был в сети" (last_seen)

**Это фундамент** — без него не работает ни один другой модуль.

---

### ⚙️ Модуль `settings`
**Таблицы:** `chat_settings`, `privacy`, `app_language`

**Для чего нужен:**
- Смена темы приложения (светлая / тёмная / системная)
- Смена размера текста
- Включение / выключение уведомлений
- Настройки приватности:
  - Кто видит номер телефона
  - Кто видит фото профиля
  - Кто видит "был онлайн"
  - Автоудаление сообщений через N дней
  - Двухфакторная аутентификация
  - Включение Safe Mode (режим шифрования)
- Язык приложения
- Автоперевод входящих сообщений

---

### 👥 Модуль `contacts`
**Таблицы:** `contact`

**Для чего нужен:**
- Добавление контакта по номеру телефона или username
- Просмотр списка всех контактов
- Удаление контакта
- Присвоение контакту кастомного имени (например "Вася с работы")
- Блокировка пользователя
- Разблокировка пользователя
- Проверка — заблокирован ли пользователь

---

### 💬 Модуль `chats`
**Таблицы:** `conversation`, `conversation_member`

**Для чего нужен:**
- Создание личного чата между двумя пользователями
- Создание группового чата
- Создание канала
- Добавление участников в группу / канал
- Удаление участников
- Назначение ролей (владелец / админ / участник)
- Изменение названия, фото, описания группы
- Генерация ссылки-приглашения
- Вступление по ссылке
- Выход из группы
- Мьют (отключение уведомлений) на время или навсегда
- Отслеживание последнего прочитанного сообщения (для счётчика непрочитанных)
- Удаление чата

---

### ✉️ Модуль `messages`
**Таблицы:** `message`, `message_status`

**Для чего нужен:**
- Отправка текстового сообщения
- Отправка сообщения с вложением (фото / видео / аудио / файл / стикер)
- Ответ на конкретное сообщение (reply)
- Пересылка сообщения в другой чат
- Редактирование отправленного сообщения
- Удаление сообщения (у себя / у всех)
- Статусы доставки: отправлено → доставлено → прочитано
- Сообщения с таймером самоуничтожения
- Загрузка истории сообщений (пагинация)
- Поиск по сообщениям

---

### 📎 Модуль `attachments`
**Таблицы:** `attachment`

**Для чего нужен:**
- Загрузка фото на сервер
- Загрузка видео на сервер
- Загрузка аудиосообщения
- Загрузка любого файла (PDF, ZIP и т.д.)
- Генерация миниатюры (thumbnail) для фото и видео
- Хранение размеров фото/видео (ширина, высота)
- Хранение длительности аудио/видео
- Хранение ключа шифрования файла (отдельный AES ключ на каждый файл)
- Скачивание файла получателем

---

### 😀 Модуль `reactions`
**Таблицы:** `reaction`

**Для чего нужен:**
- Поставить реакцию (эмодзи) на сообщение
- Убрать реакцию
- Просмотр всех реакций на сообщение
- Просмотр кто поставил какую реакцию

---

### 📞 Модуль `calls`
**Таблицы:** `call`, `call_participant`

**Для чего нужен:**
- Инициация голосового звонка
- Инициация видеозвонка
- Групповой звонок
- Принять звонок
- Отклонить звонок
- Завершить звонок
- Отслеживание статуса: звонит / активен / завершён / пропущен / отклонён
- Запись времени начала и конца (для показа длительности)
- История звонков
- Safe Mode в звонках (зашифрованный аудио/видео поток)

**Дополнительно нужен WebRTC сервер (TURN/STUN)** — отдельная инфраструктура для передачи медиапотока.

---

### 📖 Модуль `stories`
**Таблицы:** `story`, `story_view`

**Для чего нужен:**
- Публикация истории (фото или видео)
- Автоматическое удаление через 24 часа
- Просмотр историй контактов
- Отметка "кто посмотрел"
- Просмотр списка тех кто посмотрел твою историю
- Удаление истории вручную до истечения 24 часов

---

### 🔐 Модуль `encryption` (Safe Mode)
**Таблицы:** `safe_mode_session`, `safe_mode_key_share_log`, `safe_mode_ui_state`

**Для чего нужен:**
- Включение / выключение Safe Mode в настройках
- Генерация нового safe-ключа (AES-256) при включении
- Показ ключа пользователю (один раз — как seed-фраза)
- Сохранение зашифрованной версии ключа на сервере (сервер не может его прочитать)
- Синхронизация ключа между устройствами одного пользователя
- Отображение fingerprint ключа (первые 8 символов) в UI
- Поле ввода ключа в правом верхнем углу интерфейса
- Если ключ не введён — текст сообщений показывается как мусор ("a7Fk2#Xp...")
- Если ключ введён — расшифровка происходит локально на устройстве
- Зашифрованные фото / видео / аудио без ключа не открываются
- Звонки без ключа — тишина и чёрный экран
- Лог: кому и как передал ключ (QR-код / копия / ссылка / NFC)
- Автоблокировка — через N минут без активности ключ "забывается"
- Отзыв доступа (если поделился ключом и хочет закрыть)

## 🌐 ШАГ 3 — WEBSOCKET СЕРВЕР

**Что это такое:**
Постоянное соединение между приложением и сервером. В отличие от обычного HTTP, сервер сам отправляет данные клиенту в реальном времени без запроса.

**Для чего нужен:**
- Мгновенная доставка новых сообщений
- Обновление статуса "онлайн" / "оффлайн" в реальном времени
- Показ "печатает..." когда собеседник пишет
- Обновление статуса сообщения (доставлено / прочитано) без перезагрузки
- Уведомления о входящих звонках
- Обновление реакций в реальном времени

**Технологии:** Socket.IO или нативный WebSocket

## 🖥️ ШАГ 5 — МЕДИА СЕРВЕР

**Что это такое:**
Отдельный сервер для хранения и раздачи файлов (фото, видео, аудио).

**Для чего нужен:**
- Принимает загружаемые файлы от клиента
- Шифрует файлы перед сохранением
- Генерирует миниатюры для фото и видео
- Раздаёт файлы получателям по зашифрованным URL
- Автоматически удаляет файлы из сообщений с таймером
- Удаляет истории через 24 часа

**Технологии:** MinIO (self-hosted S3) или AWS S3 + CDN

## 🔑 ВАЖНЫЕ ПРИНЦИПЫ

**Безопасность:**
- Пароли хранятся только в виде хэша (bcrypt / argon2) — никогда в открытом виде
- Все сообщения зашифрованы E2E — сервер не может прочитать переписку
- Safe Mode добавляет второй слой — даже взломанный аккаунт не открывает сообщения
- Ключи шифрования файлов хранятся зашифрованными публичным ключом получателя

**Производительность:**
- Индексы в базе данных уже созданы для самых частых запросов
- Сообщения загружаются постранично (не все сразу)
- Миниатюры генерируются на сервере чтобы не грузить интернет пользователя
